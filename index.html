Phase 1: The Nervous System (API Fundamentals)

APIs act as the delivery network connecting our Apps Script (Command Center) to external databases.

1. The Transport Mechanisms
  • GET (Read): Pulls data. Requires UrlFetchApp.fetch(url) followed by JSON.parse() to rebuild the object locally.
  • POST (Create): Pushes data. Requires flattening the object into transportable text using JSON.stringify(data).
2. The Envelope (Headers & Options)
  • Content-Type: Must declare "contentType": "application/json". Without this metadata, the server rejects the payload (Status 415).
  • The Black Box Protocol: Setting "muteHttpExceptions": true prevents silent system crashes and forces the server to return its internal error logs for diagnostics.
3. Critical Diagnostic Codes
  • 201 Created: The gold standard for a successful POST. The database registered and built the entry.
  • 400 Bad Request: Syntax failure (check quotes/commas).
  • 401 Unauthorized: Missing your "Digital Keys" (Authentication).
4. Mock APIs (The Sandbox)

We test logic on Mock APIs (like jsonplaceholder) to validate our transport code. They return a 201 status to prove the truck arrived, but they immediately delete the payload to protect server space. Never test raw code on the live Nexus Database.
